# Glossary
**Adaptive Learning System**

**Version:** 1.0  
**Last Updated:** January 2025  
**Target Audience:** All Users

## Overview

This glossary provides definitions for terms, concepts, and features used throughout the Adaptive Learning System. Use this reference to understand the specialized vocabulary and technical concepts that enhance your learning or teaching experience.

---

## A

### Adaptive Algorithm
The core artificial intelligence system that personalizes learning experiences based on individual student performance, preferences, and learning patterns. Uses Bayesian Knowledge Tracing to continuously adjust content difficulty and recommendations.

### Adaptive Assessment
Evaluation method where question difficulty and type automatically adjust based on student responses. Provides more accurate competency measurement by tailoring questions to the student's current ability level.

### API (Application Programming Interface)
A set of protocols and tools that allow the Adaptive Learning System to communicate with other software applications, such as Learning Management Systems (LMS) or third-party educational tools.

### Assignment
A learning task created by educators that students must complete, such as coding challenges, quizzes, or projects. Can include automated grading, peer review, and adaptive difficulty adjustment.

### At-Risk Student
A learner identified by the system's predictive analytics as having a high probability of struggling or dropping out, based on engagement patterns, performance trends, and other behavioral indicators.

---

## B

### Bayesian Knowledge Tracing (BKT)
A probabilistic algorithm used to model and track student knowledge over time. BKT estimates the probability that a student has mastered a particular skill based on their performance on related activities.

### Bookmark
A feature that allows users to save and quickly return to specific lessons, exercises, or content sections for future reference or review.

---

## C

### Coding Challenge
Interactive programming exercises where students write code to solve specific problems. Includes automated testing, immediate feedback, and difficulty adaptation based on student competency.

### Competency
A specific skill or knowledge area that students are expected to master, such as "array manipulation" or "binary tree traversal." The system tracks competency levels on a 0-10 scale.

### Competency Map
Visual representation showing a student's mastery levels across different skills and knowledge areas, helping identify strengths and areas for improvement.

### Competency Score
Numerical rating (0-10) indicating a student's mastery level of a specific skill, calculated using Bayesian Knowledge Tracing based on performance data.

### Content Management System (CMS)
Tools that allow educators to create, organize, publish, and maintain learning materials such as lessons, assignments, and assessments.

---

## D

### Dashboard
The main interface that provides an overview of progress, recommendations, and key information for both students and educators. Customized based on user role and preferences.

### Data Structures
Fundamental computer science concepts for organizing and storing data efficiently, including arrays, linked lists, trees, graphs, stacks, and queues.

### Difficulty Adaptation
System capability to automatically adjust the complexity of learning content based on student performance and competency levels.

---

## E

### Engagement Analytics
Metrics that measure student participation and interaction with the learning platform, including time spent, activities completed, and collaboration levels.

### Educator Dashboard
Specialized interface for instructors that provides class management tools, student analytics, content creation capabilities, and communication features.

---

## F

### FERPA (Family Educational Rights and Privacy Act)
U.S. federal law that protects student educational records and gives parents/students rights over their educational information. The system complies with FERPA requirements.

### Forum
Discussion area where students and educators can ask questions, share knowledge, collaborate on problems, and engage in academic conversations organized by topic.

---

## G

### GDPR (General Data Protection Regulation)
European Union regulation governing data protection and privacy. The system complies with GDPR requirements for users in applicable regions.

### Goal Setting
Feature that allows students to create specific, measurable learning objectives with timelines and success criteria. The system tracks progress and provides recommendations.

### Gradebook
Tool for educators to view, manage, and export student grades and performance data, with integration capabilities for external Learning Management Systems.

---

## H

### Help System
Integrated support feature providing contextual assistance, searchable documentation, video tutorials, and access to support resources without leaving the learning environment.

### Hint System
Progressive assistance feature that provides increasingly specific guidance to help students solve problems without giving away complete solutions.

---

## I

### Interactive Lesson
Learning module that combines instructional content with embedded exercises, code examples, and immediate feedback to create an engaging educational experience.

### Integration
Connection between the Adaptive Learning System and external platforms such as Learning Management Systems, enabling data sharing and unified workflows.

---

## L

### Learning Analytics
Data analysis techniques used to understand and optimize learning processes, including performance tracking, pattern recognition, and predictive modeling.

### Learning Management System (LMS)
External educational platform (such as Canvas, Blackboard, or Moodle) that can integrate with the Adaptive Learning System for seamless course management.

### Learning Path
Personalized sequence of learning activities recommended by the adaptive algorithm based on student competency, goals, and preferences.

### Learning Streak
Consecutive days of active learning engagement, tracked as a motivational metric and displayed in student progress dashboards.

---

## M

### Mastery
Achievement of high competency (typically 8.0+ on the 10-point scale) in a specific skill or knowledge area, indicating thorough understanding and ability to apply concepts.

### Micro-Competency
Granular skill component within a larger topic area. For example, "array creation," "array traversal," and "array modification" are micro-competencies within the broader "arrays" competency.

### Mobile App
Native application for smartphones and tablets that provides access to learning content, progress tracking, and communication features with offline capabilities.

---

## O

### Offline Mode
Mobile app feature that allows students to download content and continue learning without an internet connection, with automatic synchronization when connectivity is restored.

---

## P

### Peer Assessment
Collaborative evaluation process where students review and provide feedback on each other's work, promoting critical thinking and communication skills.

### Personalization
Customization of learning experiences based on individual student characteristics, including learning style, competency level, goals, and preferences.

### Predictive Analytics
Advanced data analysis that forecasts future student outcomes, such as success probability or risk of dropping out, enabling proactive interventions.

### Progress Tracking
Continuous monitoring and visualization of student advancement through learning objectives, competency development, and goal achievement.

---

## Q

### Quiz
Assessment tool with various question types (multiple choice, short answer, code analysis) that can adapt difficulty based on student responses and provide immediate feedback.

---

## R

### Recommendation Engine
AI system component that suggests appropriate learning activities, resources, and next steps based on student competency, performance patterns, and learning objectives.

### Remediation
Additional learning support provided to students who are struggling with specific concepts, including alternative explanations, extra practice, and targeted resources.

---

## S

### Single Sign-On (SSO)
Authentication method that allows users to access the Adaptive Learning System using credentials from their institution's identity management system.

### Study Group
Collaborative learning feature that enables students to form groups for peer support, discussion, and joint problem-solving activities.

### Synchronization
Process of keeping data consistent across multiple devices and platforms, ensuring that progress and content are up-to-date regardless of access method.

---

## T

### Time Complexity
Measure of how algorithm execution time increases with input size, an important concept in computer science education covered by the system.

---

## U

### Universal Design
Approach to creating learning experiences that are accessible to users with diverse abilities, including visual, motor, and cognitive accommodations.

### User Role
Classification that determines access permissions and available features, including Student, Educator, Administrator, and other specialized roles.

---

## V

### Video Tutorial
Educational video content that provides step-by-step guidance on using platform features or understanding learning concepts, with captions and transcripts for accessibility.

---

## W

### Webhook
Automated notification system that sends real-time data updates to external systems when specific events occur in the Adaptive Learning System.

## Technical Terms

### Algorithm
Step-by-step procedure for solving a problem or completing a task, a fundamental concept in computer science and programming education.

### API Key
Unique identifier used to authenticate and authorize access to the system's Application Programming Interface for third-party integrations.

### Array
Basic data structure that stores elements in a sequential, indexed format, one of the foundational concepts taught in the system.

### Binary Tree
Hierarchical data structure where each node has at most two children, commonly used in computer science algorithms and data organization.

### Debugging
Process of finding and fixing errors in computer code, supported by the system's integrated development environment and feedback tools.

### Graph
Data structure consisting of nodes (vertices) connected by edges, used to represent relationships and solve complex algorithmic problems.

### IDE (Integrated Development Environment)
Built-in code editor with features like syntax highlighting, error detection, and code execution capabilities for programming exercises.

### Linked List
Linear data structure where elements are stored in nodes that contain data and references to the next node in the sequence.

### Queue
First-In-First-Out (FIFO) data structure where elements are added at one end and removed from the other, like a line of people waiting.

### Recursion
Programming technique where a function calls itself to solve smaller instances of the same problem, often used in tree and graph algorithms.

### Stack
Last-In-First-Out (LIFO) data structure where elements are added and removed from the same end, like a stack of plates.

### Syntax Highlighting
Visual formatting feature that uses colors and fonts to distinguish different parts of code, making it easier to read and understand.

## Educational Terms

### Bloom's Taxonomy
Educational framework for categorizing learning objectives and cognitive skills, used to design appropriate learning activities and assessments.

### Constructivism
Learning theory emphasizing that students actively build their own understanding through experience and reflection, supported by the system's interactive approach.

### Differentiated Instruction
Teaching approach that tailors instruction to meet individual student needs, automated through the system's adaptive algorithms.

### Formative Assessment
Ongoing evaluation during the learning process to provide feedback and guide instruction, as opposed to summative assessment at the end.

### Metacognition
Awareness and understanding of one's own thought processes, promoted through the system's reflection tools and progress visualization.

### Scaffolding
Temporary support structure that helps students achieve learning goals, gradually removed as competency increases, automated by the adaptive system.

### Zone of Proximal Development (ZPD)
Range of tasks a student can complete with guidance but not independently, targeted by the system's difficulty adaptation algorithms.

## Usage Examples

### Example: Understanding Competency Scores
```
Competency Score: 7.2/10 in "Binary Trees"
Interpretation: Student has proficient understanding
Next Steps: Ready for advanced tree algorithms
Recommendation: Try balanced tree implementations
```

### Example: Reading Learning Analytics
```
Learning Velocity: +1.8 points/week
Meaning: Student is improving competency by 1.8 points weekly
Context: Above average improvement rate
Action: Continue current learning approach
```

### Example: Interpreting Adaptive Recommendations
```
Recommendation: "Review linked list basics before attempting trees"
Reason: Prerequisite competency below threshold (5.2/10)
Benefit: Stronger foundation will improve tree learning success
Timeline: Estimated 2-3 days of review needed
```
